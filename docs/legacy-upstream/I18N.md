## Cómo extender al admin

```ts
// 1. Agrega namespace en lib/i18n/dict.ts
export const dict = {
  es: { landing: {...}, common: {...}, admin: { "dashboard.title": "Panel" } },
  en: { landing: {...}, common: {...}, admin: { "dashboard.title": "Dashboard" } },
};

// 2. Server component:
const locale = await getServerLocale();
return <h1>{t("admin", "dashboard.title", locale)}</h1>;
```

Si una key falta en `en`, hace fallback automático a `es`.

## Backend Laravel

`lang/es/validation.php` traduce todos los mensajes de validación de
formularios. Cuando se llame `$request->validate(...)` y falle, los
errores vienen en español:

```json
{
  "errors": {
    "email": ["El campo email debe ser un correo electrónico válido."],
    "password": ["El campo contraseña es obligatorio."]
  }
}
```

Atributos amigables:

```php
'attributes' => [
    'email' => 'correo',
    'password' => 'contraseña',
    'starts_at' => 'fecha y hora',
    'service_id' => 'servicio',
    'barber_id' => 'barbero',
    // ...
],
```

Para añadir EN: copiar `lang/es/validation.php` a `lang/en/validation.php`
y traducir. `App::setLocale($lang)` cambia activo según preferencia del
usuario o header `Accept-Language`.
